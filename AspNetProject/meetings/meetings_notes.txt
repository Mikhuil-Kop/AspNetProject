--- Созвон 12/03/2024 ---
* Обсудили базовую структуру проекта
    а) Разрабатываем систему мониторинга нагрузки электрических счетчиков
    б) В качестве входных данных имеем MsSQL БД с данными о показания счетчиков (у Александра есть схема БД)
    в) Будет нужна NoSQL БД для хранения логов и стейтов "счетчиков"
    г) Сами счетчики будем эмулировать через REST сервис для избавления от привязки к реальным приборам
    д) С эмуляторов многопоточно будут сниматься показания (как - обсуждаемый вопрос)
    е) Со стороны клиента будет исходить запрос на создание нагрузки на эмуляторы
    ж) Через Rabbit будет проиходить отправка запросов на эмуляторы в конкуренции с фоновой нагрузкой по шедулеру
    з) На клиенте будет отрисовываться мониторинг состояния сервера, на котором хостится MsSQL БД
* Есть понимание, какие базовые сущности должны быть в проекте, какими сервисами будет строится взаимодействие между ними.
  Можно присупать к схематичной отрисовке архитектуры
